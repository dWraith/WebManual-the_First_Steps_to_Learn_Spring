<div class="title">Что такое MySQL?</div>
<div class="text">
<br>MySQL – небольшой, компактный многопоточный сервер баз данных. MySQL характеризуется большой
скоростью, устойчивостью и легкостью в использовании.
<br>MySQL был разработан компанией TcX для внутренних нужд, которые заключались в быстрой обработке очень
больших баз данных. Компания утверждает, что использует MySQL с 1996 года на сервере с более чем 40 БД,
которые содержат 10,000 таблиц, из которых более чем 500 имеют более 7 миллионов строк.
<br>MySQL является идеальным решением для малых и средних приложений. Исходные тексты сервера
компилируются на множестве платформ. Наиболее полно возможности сервера проявляются на Unix-серверах,
где есть поддержка многопоточности, что дает значительный прирост производительности. В варианте под
Windows, <br>MySQL может запускаться как сервис Windows NT или как обычный процесс на Windows 95/98.
На текущий момент MySQL все еще в стадии разработки, хотя версии 3.22 полностью работоспособны.
<br>MySQL-сервер является бесплатным для некоммерческого использования. Иначе необходимо приобретение
лицензии, стоимость которой составляет 190 EUR.
</div>
<h1>Возможности MySQL</h1>
<div class="text">
MySQL поддерживает язык запросов SQL в стандарте ANSI 92, и кроме этого имеет множество расширений к
этому стандарту, которых нет ни в одной другой СУБД.
<br>Краткий перечень возможностей MySQL.
<br>1. Поддерживается неограниченное количество пользователей, одновременно работающих с базой данных.
<br>2. Количество строк в таблицах может достигать 50 млн.
<br>3. Быстрое выполнение команд. Возможно MySQL самый быстрый сервер из существующих.
<br>4. Простая и эффективная система безопасности.
<br>MySQL действительно очень быстрый сервер, но для достижения этого разработчикам пришлось пожертвовать
некоторыми требованиями к реляционным СУБД. В MySQL отсутствуют:
<br>1. Поддержка вложенных запросов, типа SELECT * FROM table1 WHERE id IN (SELECT id FROM table2).
Утверждается, что такая возможность будет в версии 3.23.
<br>2. Не реализована поддержка транзакций. Взамен предлагается использовать LOCK/UNLOCK TABLE.
<br>3. Нет поддержки внешних (foreign) ключей.
<br>4. Нет поддержки триггеров и хранимых процедур.
<br>5. Нет поддержки представлений (VIEW). В версии 3.23 планируется возможность создавать представления.
</div>
